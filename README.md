## 🧠 Why TrustChain?

AI hallucination is one of the most critical unsolved problems in production AI systems. When autonomous AI agents read outputs from other agents, a single hallucination from **Agent A** can silently corrupt the decision pipeline of **Agent B** — causing compounding errors across an entire agentic workflow.

**TrustChain** is a rigorous, real-world OpenEnv reinforcement learning environment that trains and evaluates agents to act as a **verification layer** in multi-agent systems. The agent receives claims generated by a simulated "Reporter Agent" (alongside optional context), and must decide whether to:

- ✅ `accept` — the claim is factually grounded and safe to propagate
- ❌ `reject` — the claim is demonstrably false or directly contradicts context
- 🔍 `verify` — the claim is ambiguous, uses absolute language on partial evidence, or requires external confirmation

This maps directly to real-world use cases: **content moderation pipelines, automated fact-checking systems, agentic AI safety layers**, and **LLM output auditing**.

## 📐 Action & Observation Spaces

### Action Space — `TrustchainAction`

| Field | Type | Description |
|---|---|---|
| `decision` | `Literal["accept", "reject", "verify"]` | The agent's verdict on the incoming claim |

- **`accept`** — For claims that are factually confirmed by context or general knowledge
- **`reject`** — For claims that are factually wrong or directly contradict the provided context
- **`verify`** — For claims that are ambiguous, use absolute language with partial evidence, or carry epistemic risk

### Observation Space — `TrustchainObservation`

| Field | Type | Description |
|---|---|---|
| `claim` | `str` | The statement submitted by the Reporter Agent for verification |
| `context` | `Optional[str]` | Contextual data provided for cross-referencing (may be `None`) |
| `difficulty` | `str` | `"easy"`, `"medium"`, or `"hard"` |
| `feedback` | `str` | Per-step feedback on the agent's previous decision |

---

## 🏆 Reward Function

The reward function is designed to provide **meaningful learning signals**, not just sparse binary feedback:

```python
if decision == ground_truth:
    reward = 1.0   # Perfect: correct decision
elif decision == "verify" and difficulty == "easy":
    reward = 0.3   # Partial: over-cautious, but not harmful
elif difficulty == "hard" and ground_truth == "verify":
    reward = 0.5   # Partial: attempted a decision on genuinely ambiguous claim
else:
    reward = 0.0   # Incorrect: wrong verdict
```

This three-tier system incentivizes genuine reasoning over random guessing, while rewarding **epistemic caution** on ambiguous hard tasks.

### 🟢 Easy — Pure Knowledge (6 tasks)
Clear factual claims with no context. Tests world knowledge grounding.

| # | Claim | Truth |
|---|---|---|

| 5 | "The speed of light in a vacuum is ~299,792 km/s." | `accept` |
| 6 | "Humans have 46 chromosomes (23 pairs)." | `accept` |

### 🟡 Medium — Context-Based Reasoning (6 tasks)
Claims with supporting context. Tests reading comprehension and mathematical validation.

| # | Context | Claim | Truth |
|---|---|---|---|
| 7 | Q2 = $10M, Q3 = $12M | "Revenue increased by 20% in Q3." | `accept` |
| 8 | Acquisition used $5M cash + $2M stock | "Company acquired using only cash." | `reject` |
| 9 | API limit: 6000 req/hour | "Rate limit is 100 requests/minute." | `accept` |
| 10 | Race condition in auth component | "Bug caused by memory leak in core." | `reject` |
| 11 | Patient showed improvement in 3/5 trials | "Treatment is effective for all patients." | `reject` |
| 12 | Server handles 10K connections with 500ms p99 | "Server efficiently handles 10K concurrent users." | `accept` |

### 🔴 Hard — Epistemic Reasoning (8 tasks)
Ambiguous claims requiring nuanced judgment. 5/8 require `verify`.

| # | Claim | Truth | Challenge |
|---|---|---|---|
| 13 | SOC2 audit: 90% controls pass, logging pending | `verify` | Absolute claim on partial audit |
| 14 | CEO "looking into hybrid options" | `reject` | Explicit promise vs. exploration |
| 15 | AES-256 "unbreakable" | `verify` | Absolute language — quantum risk |
| 16 | v1/users returns `password_hash` | `reject` | Hash ≠ plaintext password |
| 17 | Drug showed "promising results" in Phase 2 | `verify` | Phase 2 ≠ proven safety |
| 18 | Settlement reached "without admitting liability" | `verify` | Legal ambiguity |
| 19 | Model achieves 94% accuracy overall on biased dataset | `verify` | Subgroup performance unknown |
| 20 | Patch closes RCE in 99% of deployments | `verify` | 1% unpatched = still vulnerable |

## 🚀 Quick Start

### Prerequisites

```bash
# Install uv (Python package manager)
curl -LsSf https://astral.sh/uv/install.sh | sh

# Clone the repository
git clone https://github.com/HariniPrithiyangara/Trustchain-openenv.git
cd Trustchain-openenv
```

### Run Locally (Native)

```bash
uv sync
uv run python -m openenv.server.app
# Server available at http://localhost:8000
```

### Run with Docker

```bash
docker build -t trustchain-env:latest .
docker run -p 8000:8000 trustchain-env:latest
```

### Run the Baseline Inference Script

```bash
export API_BASE_URL="https://router.huggingface.co/v1"
export HF_TOKEN="<YOUR_HF_TOKEN>"
export MODEL_NAME="meta-llama/Llama-3.3-70B-Instruct"

# Optional: use local docker image instead of live Space
export LOCAL_IMAGE_NAME="trustchain-env:latest"

uv run python inference.py
```

## 🤖 Inference Script Design

The `inference.py` script implements a **two-pass Chain-of-Thought (CoT)** prompting strategy:

1. **Pass 1 — Reasoning:** Model reasons through the claim, context, and numerical math step-by-step internally
2. **Pass 2 — Extraction:** A secondary model call extracts the final single-word decision (`accept`/`reject`/`verify`) from the reasoning trace

This dramatically improves accuracy on numerical and logical tasks (e.g., correctly computing `6000 req/hr ÷ 60 = 100 req/min → accept`).
