# TrustChain Environment

An advanced, real-world OpenEnv environment simulating an AI content verification and fact-checking system.

## Motivation & Description

AI hallucination is a significant problem. In an ecosystem where autonomous AI agents read outputs from other agents, a hallucination generated by Agent A can seamlessly pollute the decision-making pipeline of Agent B, leading to catastrophic compounding errors.

**TrustChain** simulates a realistic agentic fact-checking layer. It feeds your agent with 'claims' generated by a mock "Reporter Agent" alongside optional 'contexts'. The TrustChain environment teaches your agent to accurately decide whether to `accept`, `reject`, or `verify` (request manual intervention/more info) the incoming claims. The underlying objective is to align agents to be skeptical of unverified assumptions—learning to detect contradictions, absolute falsehoods, and nuanced ambiguities.

## Setup Instructions

### Pre-requisites

Ensure you have Docker installed and the `uv` package manager setup.

```bash
# Clone the repository
git clone <repository_url>
cd openenv

# Install dependencies using uv
uv sync
```

### Running Locally

You can launch the environment natively using:
```bash
uv run --project . server
```
Or start a containerized environment (highly recommended for deployment matching):
```bash
docker build -t trustchain-env:latest .
docker run -p 8000:8000 trustchain-env:latest
```

## Action & Observation Spaces

### Action Space
`TrustchainAction`: A single model field defining the user action.
- `decision` (Literal["accept", "reject", "verify"]) - The decision on the claim.
  - `accept` - For obviously true or context-verified truths.
  - `reject` - For obviously false, contradictory, or clearly hallucinated claims.
  - `verify` - For claims that lack evidence in context, are ambiguous, or carry high risk without profiling.

### Observation Space
`TrustchainObservation`:
- `claim` (str) - The statement or observation supplied by the reporter agent.
- `context` (Optional[str]) - Contextual data/sources given for cross-referencing.
- `difficulty` (str) - Easy, medium, or hard.
- `feedback` (str) - Context regarding whether the last step was correct.

### Reward Function
The agent is programmatically evaluated:
- Reward `1.0` if the decision perfectly aligns with the deterministic truth condition.
- Reward `0.0` otherwise.

## Tasks and Difficulty Ranges

The environment consists of 12 sequential tasks seamlessly blending `easy`, `medium`, and `hard` claims, grouped in an escalating curriculum for complete task exploration in a single episode run.

1. **Easy (Knowledge Facts)** 
   - Basic factual information validation without complex context. (e.g. "Earth has 2 moons." -> reject).
   - *Challenges:* General grounding. 
2. **Medium (Context-Based Parsing)**
   - Extractive and logic-based context reading. (e.g. Context mentions missing parts; Claim asserts full completion -> reject).
   - *Challenges:* In-context reading comprehension.
3. **Hard (Nuance and Verification)**
   - Ambiguous constraints, incomplete audits, and lack of absolute truth.
   - *Challenges:* Detecting when to output "verify" instead of guessing true or false.

## Baseline Run

To test a baseline LLM (e.g., Qwen/Qwen2.5-72B-Instruct) on this agentic task:
```bash
export API_BASE_URL="<YOUR_LLM_API>"
export HF_TOKEN="<YOUR_API_KEY>"
export MODEL_NAME="Qwen/Qwen2.5-72B-Instruct"

# Using a local docker image (Optional, but replicates standard OpenEnv test pipeline)
export IMAGE_NAME="openenv-env:latest"

# Run inference
uv run python inference.py
```

### Reproducible Baseline Scores
Running tests on `Qwen2.5-72B-Instruct` achieves an approximate baseline score of **~0.66 (8/12)**, primarily struggling with the "Hard" tasks involving nuanced assumptions and partial facts. Running a smarter frontier model will score significantly closer to **1.0**.
